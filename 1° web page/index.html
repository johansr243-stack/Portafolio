Conceptos HTML & CSS

Los primeros pasos en HTML & CSS son fundamentales ya que presentan la estructura con la que empezamos a escribir codigo que permite desarrollar paginas web interactivas, a continuación presentamos los primeros temas vistos y sus definiciones.

Box Model

Determina cómo se calcula el tamaño total de un elemento y cómo interactúa con los demás. Básicamente, considera que cada etiqueta HTML es una caja invisible con cuatro capas: Content (el contenido), Padding (espacio interno), Border (el borde) y Margin (espacio externo).

Un detalle curioso

Por defecto, CSS tiene una forma "traicionera" de sumar tamaños. Si defines una caja de 300px de ancho y le añades 20px de padding, ¡la caja medirá 340px en total! Para evitar este dolor de cabeza y que las medidas sean exactas, los desarrolladores usamos este "truco" universal al principio de cada archivo CSS: box-sizing: border-box;. Esto hace que el padding y el borde se queden dentro del ancho que tú definiste

Display

Define el espacio a ocupar de los elementos dentro de una caja.

Un detalle curioso

Cada elemento HTML tiene un valor de display por defecto. Por ejemplo, un div siempre es block y un span siempre es inline. Con esta propiedad, tú tienes el poder de romper esas reglas naturales.

Flex Box

Es la dirección en la que se ubican los elementos dentro de una caja.

Un detalle curioso

A diferencia de otros métodos, Flexbox tiene el concepto de Ejes. El Main Axis (Eje principal): Por defecto es horizontal (fila). El Cross Axis (Eje cruzado): Por defecto es vertical. Lo "loco" es que si cambias la dirección a columna (flex-direction: column), ¡los ejes se intercambian! Lo que antes alineaba horizontalmente, ahora lo hará verticalmente. Es como girar el mando de una televisión; las reglas siguen siendo las mismas, pero la orientación cambia.

Measurements

Son las propiedades de valor que le damos de acuerdo al tamaño que se quiere aplicar al elemento.

Un detalle curioso

Mucha gente se confunde entre em y rem. La r de rem viene de "Root" (raíz). Siempre vale lo mismo en toda la página (usualmente 16px). em, en cambio, es "hereditario". Si pones 2em a un texto dentro de una caja que ya tiene 2em, el texto se hace gigante porque se multiplica por su padre. Tip de pro: Usa siempre rem para fuentes. Así, si un usuario aumenta el tamaño de letra en su navegador por accesibilidad, tu web se adaptará perfectamente sin romperse.

Selectors

Son las etiquetas que llaman los elementos que queremos aplicar en CSS.

Un detalle curioso

¿Qué pasa si a un mismo botón le dices que sea rojo por su etiqueta, pero verde por su clase? CSS tiene un sistema de puntos para decidir quién gana. Imagina que es un sistema de rangos: ID (#): Es el General (vale 100 puntos). Clase (.): Es el Sargento (vale 10 puntos). Etiqueta: Es el Soldado (vale 1 punto). Si hay conflicto, el General siempre gana al Sargento, y el Sargento al Soldado. Por eso, aunque escribas el color de la etiqueta al final del archivo, si hay una clase que dice lo contrario, la clase ganará por "rango".

Forms

Espacios de campo donde se insertas datos para obtener información sobres los usuarios.

Un detalle curioso

Existe una "conexión invisible" vital para la accesibilidad: el atributo for del label debe ser igual al id del input. Si haces esto bien, cuando el usuario haga clic en el texto de la etiqueta, el cursor saltará automáticamente al campo de entrada. Esto es fundamental para personas que usan lectores de pantalla o simplemente para mejorar la experiencia en móviles, donde los dedos necesitan objetivos más grandes para tocar.
